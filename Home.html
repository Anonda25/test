1. What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface used to handle the structure and content of web pages. DOM which converts HTML documents into a tree structure, where each element (eg, tag, attribute, text) is represented as a node.

3. What is event delegation in the context of the DOM, and why is it useful?

Event delegation in the context of the DOM is an efficient technique that makes the event handling process simpler and more efficient.

How event delegation works: Event delegation works based on the concept of event bubbling. Event bubbling is a process where an event first occurs at its specific target (eg, a button click) and then propagates to parent elements in a bubble.

01. Why Event Delegation is useful: Less event handler usage: Each child element has event handlers separately!

02.Dynamic Element Handling: When new child elements are dynamically added to the DOM, there is no need to add event handlers separately.
